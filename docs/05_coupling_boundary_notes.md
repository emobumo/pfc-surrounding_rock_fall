# 05 Coupling Boundary Notes

## 1. 耦合边界的作用

本项目采用 PFC2D 与 FLAC2D 的 wall-zone 耦合方法。耦合边界连接内部 PFC 离散颗粒区域与外部 FLAC2D zone 连续介质区域。

通过该边界，PFC 区域与 FLAC zone 区域能够进行力学相互作用，从而同时表达近场破坏崩落和远场应力响应。

## 2. 区域关系

当前模型的区域关系为：

```text
FLAC2D zone 远场围岩区域
┌─────────────────────────────┐
│                             │
│       PFC2D 颗粒区域         │
│   ┌─────────────────────┐   │
│   │   矿体 + 近场围岩    │   │
│   │                     │   │
│   │   矿房区域：删除颗粒 │   │
│   └─────────────────────┘   │
│                             │
└─────────────────────────────┘
```

其中：

- PFC 区域包含矿体及近场围岩；
- FLAC zone 区域包裹 PFC 区域；
- 矿房开采通过删除 PFC 区域内对应矿房的颗粒实现；
- FLAC zone 区域用于观察远区应力分布和边界影响。

## 3. 原始版本中的耦合价值

原始版本已经验证可以运行，其 wall-zone 耦合流程是后续重构的重要基础。

后续改写 PFC 部分时，应尽量保留原始版本中已经验证的耦合思路，但需要根据新的伺服后几何边界重新调整：

- 耦合墙位置；
- zone 区域范围；
- PFC 区域范围；
- zone 空区与 PFC 区域的匹配关系。

## 4. 当前耦合问题

原始版本中存在一个重要问题：

- PFC 区域未充分恢复原岩应力；
- 后期计算中可能出现上部颗粒离开耦合墙；
- 这说明 PFC 初始应力状态、耦合边界接触状态或边界几何对接还不够合理。

因此，后续不能只在原始版本上直接开挖，而需要先改进 PFC 初始应力恢复。

## 5. 伺服墙体与耦合墙的区别

### 5.1 伺服墙体

伺服墙体用于 PFC 颗粒区域原岩应力恢复阶段。

其作用是：

- 对颗粒集合施加边界加载；
- 调整墙体速度或位置；
- 使颗粒区达到目标水平应力和竖向应力；
- 形成较合理的初始应力状态。

伺服墙体是前处理阶段的加载边界。

### 5.2 耦合墙

耦合墙用于 PFC 与 FLAC zone 之间的 wall-zone 耦合。

其作用是：

- 连接 PFC 颗粒区和 FLAC zone 区；
- 在开挖和崩落计算中传递力学响应；
- 维持近场离散区与远场连续区的相互作用。

耦合墙是耦合计算阶段的接口边界。

## 6. 关键注意事项

后续改写时，需要重点解决以下问题：

1. 伺服完成后是否删除伺服墙体；
2. 耦合墙如何根据伺服后的 PFC 区域创建；
3. zone 区域是否需要重新生成；
4. wall-zone 耦合范围必须与 PFC 区域一致；
5. 上部颗粒离开耦合墙问题需要重点验证。

## 7. 当前优先解决的问题

当前阶段最重要的问题是：

> PFC 区域、伺服后边界、耦合墙和 FLAC zone 区域之间的尺寸对接。

建议后续每次修改耦合相关代码时，都检查：

- PFC 区域实际范围；
- 伺服墙体最终位置；
- 耦合墙创建范围；
- zone 区域内边界范围；
- PFC 与 zone 是否重叠或脱开；
- 开挖后颗粒是否异常离开耦合墙。
