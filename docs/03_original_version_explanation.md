# 03 Original Version Explanation

## 1. 原始版本定位

原始版本是本项目中最早保存的关键可运行版本。

该版本的主要价值不在于 PFC 原岩应力恢复是否完善，而在于它已经实现并验证了：

1. PFC2D 与 FLAC2D 的 wall-zone 耦合流程；
2. 删除矿房颗粒形成采空区的基本流程；
3. 开挖后按时间自动保存模型状态的逻辑。

因此，原始版本应作为后续重构的基准版本保存，不建议直接覆盖。

## 2. 原始版本基本流程

原始版本流程可概括为：

```text
生成 PFC 颗粒样品
→ 创建 FLAC zone 区域
→ 删除临时边界墙
→ 创建 wall-zone 耦合边界
→ 添加 PFC 接触模型
→ 施加重力并平衡
→ 删除矿房颗粒
→ 开挖后继续计算
→ 自动保存中间状态
```

## 3. 原始版本关键文件理解

### 3.1 颗粒生成文件

通常用于：

- 新建模型；
- 定义 PFC 区域几何尺寸；
- 创建颗粒生成边界；
- 生成颗粒；
- 设置颗粒密度、阻尼和初始接触模型；
- 计算至初始稳定；
- 保存初始颗粒样品。

### 3.2 zone 创建与耦合文件

通常用于：

- 读取初始颗粒样品；
- 创建 FLAC2D zone 区域；
- 设置 zone 密度、本构模型和力学参数；
- 删除 PFC 颗粒生成阶段使用的临时墙体；
- 通过 wall-zone 创建 PFC 与 FLAC 的耦合边界；
- 设置 PFC 接触模型；
- 施加重力和边界约束；
- 计算自重平衡；
- 保存耦合后的初始模型。

### 3.3 开挖与保存文件

通常用于：

- 读取耦合平衡后的模型；
- 定义矿房位置和尺寸；
- 将矿房范围内 PFC 颗粒分组；
- 删除矿房区域颗粒，形成采空区；
- 清零位移或重置相关记录量；
- 设置自动保存函数；
- 添加 FISH callback；
- 计算开挖后的响应；
- 保存最终模型。

### 3.4 总控文件

总控文件用于按顺序调用各阶段 `.dat` 文件。后续重构后仍建议保留总控文件，使完整流程能够一键运行。

## 4. 原始版本已知问题

虽然原始版本可以运行，但其 PFC 初始状态存在不足：

- PFC 颗粒区域没有充分恢复原岩应力；
- 颗粒初始应力状态不够合理；
- 后期可能出现上部颗粒离开耦合墙的现象；
- 这种现象说明 PFC 区域与耦合边界之间的初始受力状态需要改进。

## 5. 后续应保留的内容

1. wall-zone 耦合的基本建模思想；
2. FLAC zone 包裹 PFC 区域的整体结构；
3. 删除 PFC 颗粒形成采空区的开挖方式；
4. 开挖后自动保存模型状态的 FISH callback 思想；
5. 总控文件按阶段调用的组织方式。

## 6. 后续应重写的内容

1. PFC 颗粒生成后的初始应力恢复流程；
2. 伺服墙体加载与稳定判据；
3. 颗粒赋参流程；
4. 接触模型添加或切换流程；
5. 伺服完成后 PFC 区域、耦合墙和 zone 区域的尺寸对接方式。
