# 08 Common Errors

本文件用于记录 PFC2D 6.0、PFC5.0 转换、wall-zone 耦合和项目重构过程中遇到的常见问题。

## 1. PFC5.0 与 PFC2D 6.0 语法混用

### 问题

参考案例“含裂隙岩体原岩应力平衡与开挖扰动模拟”为 PFC5.0 版本，而当前项目使用 PFC2D 6.0。

如果直接复制代码，可能出现语法错误或命令行为不一致。

### 处理原则

- 先梳理代码逻辑，再转换语法；
- 不要机械复制参考案例；
- 每转换一个模块都单独测试；
- 将发现的语法差异记录到本文档。

## 2. group 名称未使用字符串

### 典型报错

```text
Bad conversion of parameter number ...
String tokens must be surrounded by " or ' characters.
```

### 可能原因

PFC2D 6.0 中部分命令的 `group` 名称需要写成字符串。

错误写法示例：

```text
wall create id 1 group 1 ...
```

建议写法示例：

```text
wall create id 1 group "1" ...
```

具体语法应以 PFC2D 6.0 实际命令要求为准。

## 3. 伺服墙体边界出现缝隙

### 现象

墙体伺服后边界连接处出现缝隙，颗粒可能从墙体连接处散出。

### 可能原因

- 四面墙体创建端点不一致；
- 墙体移动方向或速度设置不合理；
- 墙体伺服过程中边界不同步；
- 墙体交界处没有足够重叠或连接余量。

### 检查方法

- 检查四面墙体初始端点；
- 检查墙体最终位置；
- 观察颗粒是否从角部散出；
- 检查四条应力 history 是否趋于稳定。

## 4. PFC 颗粒离开耦合墙

### 现象

原始版本后期计算中，上部颗粒可能离开耦合墙。

### 当前理解

该问题可能与 PFC 颗粒区域未充分恢复原岩应力有关。

### 处理方向

- 在耦合前加入伺服墙体恢复原岩应力；
- 确认颗粒初始应力状态合理；
- 检查耦合墙位置是否与 PFC 区域对接；
- 检查 zone 区域内边界与 PFC 区域是否错位；
- 检查伺服墙体和耦合墙是否发生冲突。

## 5. 伺服墙体与耦合墙混淆

### 问题

伺服墙体和耦合墙都是 wall，但作用不同。

| 类型 | 阶段 | 作用 |
|---|---|---|
| 伺服墙体 | PFC 初始应力恢复阶段 | 对颗粒加载，恢复原岩应力 |
| 耦合墙 | PFC-FLAC 耦合阶段 | 连接 PFC 与 FLAC zone |

### 处理原则

- 明确墙体编号和分组；
- 伺服完成后决定是否删除临时墙体；
- 耦合墙应在尺寸对接明确后创建；
- 不要让伺服墙体残留并干扰 wall-zone 耦合。

## 6. zone 区域与 PFC 区域尺寸不匹配

### 现象

PFC 区域、耦合墙和 zone 区域边界错位，可能导致耦合失败或计算异常。

### 可能原因

- 伺服后 PFC 区域尺寸变化；
- 仍沿用原始版本的 zone 创建范围；
- wall-zone create 范围未同步修改；
- PFC 区域和 FLAC zone 内边界之间存在重叠或间隙。

### 检查方法

- 输出 PFC 区域边界坐标；
- 输出伺服墙体最终位置；
- 检查 zone 内边界坐标；
- 检查 wall-zone create 的 range；
- 可视化耦合边界位置。

## 7. 原始版本被误覆盖

### 问题

后续重构时，如果直接在原始版本上修改，可能丢失已验证可运行的耦合流程。

### 处理原则

- 原始版本单独保存；
- 主线改写放在 `src/current/`；
- 重大改动前先 commit；
- 关键阶段使用 tag 标记；
- 不要直接覆盖 `original_v0.1`。

## 8. 结果输出过早复杂化

### 问题

在整体流程尚未打通之前，过早整理复杂输出可能分散重点。

### 当前原则

当前优先级为：

1. PFC 原岩应力恢复；
2. PFC 与 FLAC zone 尺寸对接；
3. wall-zone 耦合恢复；
4. 删除颗粒形成采空区；
5. 流程打通；
6. 最后再完善状态保存、history 和结果图。
